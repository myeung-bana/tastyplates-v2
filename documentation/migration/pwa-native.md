## 1. Core Philosophy

Bundling a PWA into a native shell is **not** a silver bullet — it is a deliberate architectural decision. The goal is to preserve your web investment while gaining:

- App store distribution (Google Play, Apple App Store)
- Access to native device APIs (camera, biometrics, push notifications)
- Offline-first capability via service workers + native cache
- A cohesive, installable experience

> **Golden Rule:** Your web layer should remain a fully functional PWA independent of any native wrapper. The native shell augments; it never dictates.

## 2. Technology Decision Tree

```
Is your team comfortable with native code?
│
├── YES → Capacitor (recommended for Next.js)
│          Gives you fine-grained native plugin control
│
└── NO  → Is your app mostly content/display?
           │
           ├── YES → WebView Wrapper (TWA for Android / WKWebView for iOS)
           │          Lightweight, no native code required
           │
           └── NO  → Expo + Next.js (via expo-router)
                      Best DX if you plan to grow native features
```

**Recommended stack for Next.js:** `Next.js (static export) + Capacitor`

### 3.3 Key Directory Purposes

| Directory | Purpose |
|-----------|---------|
| `public/icons/` | All PWA icon sizes. Required: 192×192, 512×512. Include maskable variants. |
| `public/manifest.json` | PWA manifest. Controls install behaviour on all platforms. |
| `src/components/native/` | Components that conditionally use Capacitor APIs. Always guard with `isPlatform('hybrid')`. |
| `src/hooks/usePlatform.ts` | Single source of truth for platform detection (`web` \| `android` \| `ios`). |
| `src/lib/capacitor.ts` | Lazy-loads and initialises Capacitor plugins. Prevents SSR crashes. |
| `android/` / `ios/` | Auto-generated by Capacitor. Treat as build artefacts — commit but do not hand-edit. |

## 4. Rules of Thumb

### 4.1 Architecture

1. **Static export is mandatory.** Set `output: 'export'` in `next.config.js` for native builds. Capacitor serves static files; it cannot run a Node.js server.
2. **One codebase, two outputs.** Your `next build` should produce both a deployable web bundle and a static export consumed by the native shell.
3. **Feature-detect, not platform-detect.** Check for API availability (`'share' in navigator`) before checking the platform name.
4. **The web layer must work without the native wrapper.** Every native feature must have a graceful web fallback.
5. **Service workers are your offline contract.** Define a clear caching strategy per route type (network-first for API, cache-first for assets).

### 4.2 Performance

6. **Treat the WebView as a constrained browser.** Assume slower JavaScript execution than a desktop browser. Budget your JS payload accordingly.
7. **Lazy-load Capacitor plugins.** Never import Capacitor plugins at the top level. Use dynamic imports to avoid SSR breaks and reduce initial bundle size.
8. **Avoid re-renders on navigation.** Use React layout components and persistent state to prevent full page repaints that feel jarring in a native shell.

### 4.3 UX

9. **Respect safe areas.** Always account for the iOS notch and Android status bar using CSS `env(safe-area-inset-*)`.
10. **Remove tap delay.** Ensure `touch-action: manipulation` is applied globally.
11. **Disable text selection on interactive elements.** Users expect native-like touch interactions.
12. **Match platform navigation conventions.** Android users expect a back button; iOS users swipe back. Build accordingly.

### 4.4 Build

13. **Version your native and web builds together.** Use a single `version` field in `package.json` and propagate it to `android/app/build.gradle` and `ios/App/Info.plist` via CI.
14. **Sign builds in CI, never locally.** Store keystores and provisioning profiles as encrypted CI secrets.
15. **Keep `android/` and `ios/` in source control.** But treat them as managed output — run `npx cap sync` after every dependency change.

## 5. Do's

### Next.js Configuration

- ✅ Set `output: 'export'` and `trailingSlash: true` for static export compatibility
- ✅ Use `next/image` with `unoptimized: true` for static exports (or use a CDN loader)
- ✅ Use environment variables prefixed with `NEXT_PUBLIC_` for any client-side config
- ✅ Configure `assetPrefix` if serving assets from a CDN

### PWA Manifest & Icons

- ✅ Include `"display": "standalone"` in `manifest.json`
- ✅ Provide maskable icons (use [maskable.app](https://maskable.app) to verify safe zones)
- ✅ Include a 1024×1024 icon for App Store submission
- ✅ Set `"theme_color"` and `"background_color"` to match your splash screen
- ✅ Define `"scope"` and `"start_url"` explicitly

### Service Worker

- ✅ Use `next-pwa` or `serwist` to manage service worker generation
- ✅ Use `StaleWhileRevalidate` for API routes, `CacheFirst` for static assets
- ✅ Implement a custom offline fallback page at `/offline`
- ✅ Version your cache names (e.g., `cache-v1`) and clean up old caches on activation

### Capacitor

- ✅ Always call `await SplashScreen.hide()` manually after your app is ready
- ✅ Use `Capacitor.isNativePlatform()` to guard all native-only code
- ✅ Run `npx cap sync` after every `npm install` that adds a Capacitor plugin
- ✅ Use `@capacitor/app` to handle the hardware back button on Android
- ✅ Handle deep links via `@capacitor/app`'s `appUrlOpen` listener

### Security

- ✅ Set a strict Content Security Policy (CSP)
- ✅ Use HTTPS for all remote API calls, even in development via a tunnel
- ✅ Enable certificate pinning for sensitive apps using `@capacitor-community/http`
- ✅ Store sensitive data in `@capacitor/preferences` (not `localStorage`) on native

## 6. Don'ts

### Architecture

- ❌ **Don't use Next.js API routes as your backend in the native build.** The static export doesn't include them. Move API logic to a separate backend service.
- ❌ **Don't use `next/headers` or server-only imports** anywhere in code that reaches the client bundle.
- ❌ **Don't rely on `window` or `document` at the module level.** Always guard with `typeof window !== 'undefined'` or use `useEffect`.
- ❌ **Don't bundle your entire app as a remote URL load in production** (i.e., don't set `server.url` in `capacitor.config.ts` for production). Use local static files.

### Performance

- ❌ **Don't ship uncompressed assets.** Always enable Gzip/Brotli for web; for native, Capacitor serves from the local filesystem so keep assets lean.
- ❌ **Don't use CSS animations driven by JavaScript scroll events** — they kill performance inside WebViews. Use CSS `scroll-timeline` or Intersection Observer.
- ❌ **Don't use `position: fixed` for bottom navbars without `env(safe-area-inset-bottom)`** — it overlaps the home indicator on iOS.

### Native Bridge

- ❌ **Don't call Capacitor plugins in SSR context.** Always lazy import them inside `useEffect` or event handlers.
- ❌ **Don't hand-edit generated `android/` or `ios/` project files** except in designated extension points (`android/app/src/main/res/`, `ios/App/App/`).
- ❌ **Don't bypass the Capacitor plugin system** to call native code directly via `WKScriptMessageHandler` unless building a custom plugin properly.

### App Store Compliance

- ❌ **Don't use `iframe` to load external content** without a sandboxing policy — app store reviewers flag this.
- ❌ **Don't include any remote code execution** paths (e.g., `eval`, dynamic `<script>` injection from a server).
- ❌ **Don't request permissions at startup.** Request camera, location, and notification permissions contextually (at the moment the feature is used).

### UX

- ❌ **Don't use browser-native UI elements** (`alert()`, `confirm()`, `prompt()`). Replace with in-app modal components.
- ❌ **Don't use `<a target="_blank">` without handling it via `@capacitor/browser`** — it will try to open the system browser.
- ❌ **Don't allow rubber-band/overscroll on the root element** — it looks unpolished in a native shell. Use `overscroll-behavior: none` on `<body>`.

## 8. Native Bridge Layer Guidelines

### 8.1 Platform Detection Hook

```typescript
// src/hooks/usePlatform.ts
import { Capacitor } from '@capacitor/core';

export type Platform = 'web' | 'android' | 'ios';

export function usePlatform(): Platform {
  if (!Capacitor.isNativePlatform()) return 'web';
  return Capacitor.getPlatform() as Platform;
}
```

### 8.2 Safe Capacitor Plugin Import Pattern

```typescript
// src/lib/capacitor.ts
// Never import at module level — always lazy load inside effects or handlers

export async function shareContent(title: string, url: string) {
  if (typeof window === 'undefined') return;
  const { Capacitor } = await import('@capacitor/core');
  
  if (Capacitor.isNativePlatform()) {
    const { Share } = await import('@capacitor/share');
    await Share.share({ title, url });
  } else {
    // Web fallback
    if ('share' in navigator) {
      await navigator.share({ title, url });
    } else {
      await navigator.clipboard.writeText(url);
    }
  }
}
```

### 8.3 Safe Area CSS Variables

```css
/* globals.css */
:root {
  --safe-area-top: env(safe-area-inset-top, 0px);
  --safe-area-bottom: env(safe-area-inset-bottom, 0px);
  --safe-area-left: env(safe-area-inset-left, 0px);
  --safe-area-right: env(safe-area-inset-right, 0px);
}

/* Apply to viewport root */
html, body {
  overscroll-behavior: none;
  -webkit-tap-highlight-color: transparent;
  touch-action: manipulation;
}
```

## 10. Performance Checklist

- [ ] Lighthouse PWA score ≥ 90 before wrapping
- [ ] First Contentful Paint < 1.5s on mid-range device
- [ ] JS bundle < 300KB (compressed) for initial route
- [ ] All images are WebP/AVIF with explicit `width` and `height`
- [ ] Fonts are self-hosted (no Google Fonts CDN calls from within WebView)
- [ ] Service worker precaches critical shell assets
- [ ] No layout shift (CLS = 0) on app launch
- [ ] Capacitor plugins are code-split and lazy-loaded
- [ ] `react-query` or `swr` used for data fetching with stale-while-revalidate
- [ ] Background sync registered for form submissions

---

## 11. Security Checklist

- [ ] CSP header configured (disallow `unsafe-eval`, `unsafe-inline`)
- [ ] All API calls use HTTPS
- [ ] Sensitive tokens stored in `@capacitor/preferences` (Keychain/Keystore backed), never `localStorage`
- [ ] OAuth flows use `@capacitor/browser` (in-app browser), never embedded WebView
- [ ] Deep link validation — verify URL scheme and host before processing
- [ ] Certificate pinning enabled for production API endpoints
- [ ] ProGuard/R8 minification enabled for Android release builds
- [ ] Bitcode disabled, symbols stripped for iOS release
- [ ] No API keys or secrets in the static export or JS bundle
- [ ] App permissions reviewed — request only what is necessary
