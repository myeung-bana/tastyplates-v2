# Database Migrations

This folder contains SQL migrations and the Tastyplates backup for importing into a database.

## Files

| File | Purpose |
|------|---------|
| `tastyplates-backup.sql` | Binary pg_dump archive (custom format). **Do not** run as SQL. |
| `tastyplates-restore.sql` | Plain SQL generated from the backup. Run with `psql -f` to **import** into a database. *(Generated by the script below.)* |
| `location_taxonomy_schema.sql` | Incremental migration: location taxonomy tables. |
| `cursor_pagination_indexes.sql` | Incremental migration: indexes for cursor pagination. |
| `generate-restore-sql.sh` | Script to convert the binary backup into `tastyplates-restore.sql`. |

## Method 1: Generate plain SQL from the backup

The backup file is in **binary** format. To get runnable SQL:

1. **Generate the restore script** (requires PostgreSQL client tools or Docker):

   ```bash
   ./generate-restore-sql.sh
   ```

   This creates `tastyplates-restore.sql` in this folder.

2. **Import into a database**:

   ```bash
   psql "$DATABASE_URL" -f tastyplates-restore.sql
   ```

   Or, with explicit connection:

   ```bash
   psql -h host -U user -d dbname -f tastyplates-restore.sql
   ```

### If you don't have `pg_restore` installed

- **macOS (Homebrew):** `brew install postgresql@17` then run `./generate-restore-sql.sh`
- **Docker:** The script tries Docker automatically if `pg_restore` is missing.

## Run order for a new database

1. **Full import from backup:**  
   `psql "$DATABASE_URL" -f tastyplates-restore.sql`

2. **Then run any incremental migrations** (if they are additive and not already in the backup):  
   - `location_taxonomy_schema.sql`  
   - `cursor_pagination_indexes.sql`  

   Example:

   ```bash
   psql "$DATABASE_URL" -f location_taxonomy_schema.sql
   psql "$DATABASE_URL" -f cursor_pagination_indexes.sql
   ```

## Alternative: Restore the binary backup directly

Without converting to plain SQL, you can restore the backup with:

```bash
pg_restore -d "$DATABASE_URL" tastyplates-backup.sql
```

Use `-c` to drop objects before recreating (destructive). Omit `-c` to add into an existing database (may conflict if objects already exist).
